Разработка АСУ и HMI для системы микроклимата птичника

Проектирование управляющей логики и человеко-машинного интерфейса на базе контроллера ОВЕН СПК107 (CodeSys).

ПЛК ОВЕН CodeSys HMI Design Язык ST

Обзор проекта

Данный проект представляет собой полный цикл разработки автоматизированной системы управления микроклиматом птичника. В связи с необходимостью повышения надежности и гибкости управления, было принято решение создать собственное решение на базе российского ПЛК ОВЕН СПК107 (CodeSys v3). Хотя проект не был внедрен в эксплуатацию из-за сжатых сроков заказчика, он демонстрирует глубокие компетенции в HMI-дизайне, программировании логики и понимании отраслевых требований.

Визуализация HMI

Интерфейс был спроектирован с упором на интуитивность и удобство для оператора, позволяя легко контролировать состояние оборудования, видеть аварийные сигналы и настраивать ключевые параметры.

Главный экран (Мониторинг)

Отображение ключевых параметров (температура, влажность, статус вентиляции и обогрева).

Настройка уставок

Установка дневных/недельных режимов и предельных отклонений.

Калибровка датчиков

Интерфейс для точной настройки и калибровки температурных датчиков (T1, T2, T3).

Техническая реализация

Логика ПЛК (CodeSys, ST)

Вся управляющая логика была написана на языке ST (Structured Text) в среде CodeSys, обеспечивая чистоту кода, его читаемость и удобство сопровождения. Логика охватывала следующие модули:

  • **Управление климатом:** Модуль ПИД-регулирования для обогрева, алгоритмы ступенчатой вентиляции.
  • **Обработка аварий:** Система немедленной сигнализации и защитного отключения (например, при перегреве или сбое датчиков).
  • **Обработка ввода/вывода:** Маппинг и нормализация сигналов с аналоговых и дискретных входов.
  • **Графическая часть:** Привязка внутренних переменных к элементам HMI.

Пример структуры I/O

Структура ввода/вывода, определенная в проекте, демонстрирующая комплексное управление: 

// Управляющие выходы (DO)
DO_OBOREV_1: BOOL; // Обогреватель 1
DO_VENT_GL: BOOL;  // Главная вентиляция
DO_OKNO_1: INT;    // Положение окна 1 (аналог.)
DO_ALARM_LAMP: BOOL; // Аварийная лампа

// Входы (AI/DI)
AI_TEMP_ZONE1: REAL; // Температура в Зоне 1
DI_DOOR_LIMIT: BOOL; // Концевик двери
DI_MOTOR_FAULT: BOOL; // Ошибка мотора
AI_HUMIDITY: REAL;   // Влажность

Результаты и выводы

Ключевой вклад:

Проект подтверждает способность к полному циклу разработки АСУ: от системного анализа требований и проектирования интерфейса (HMI) до низкоуровневого программирования ПЛК (ST). Этот навык критически важен для роли ведущего инженера, который должен не только обслуживать, но и создавать/модернизировать системы.

Этот опыт лег в основу более сложного проекта по автоматизации бадьямойки Nieros, где знания в HMI и логике были успешно применены и внедрены.
