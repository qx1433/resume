WMS PWA — Складской учёт запчастей

Progressive Web Application для каталогизации и учёта запчастей на складах птицеводческого предприятия. Покрытие: 3 склада, 44 производственных зала.

Обзор проекта

Учёт запчастей на трёх складах и 44 производственных залах велся разрозненно, без единой системы. Склады расположены в зонах без стабильного Wi-Fi, что делало невозможным использование облачных решений без офлайн-поддержки. Было принято решение разработать собственное PWA-приложение, работающее полностью автономно и синхронизирующее данные при появлении связи.

Задачи

Централизация учёта

Объединить учёт запчастей across цехов и ферм в единую систему с реалтайм-синхронизацией между пользователями.

Офлайн-режим

Обеспечить полную работу без интернета — склады без Wi-Fi, мобильная связь нестабильна. Данные сохраняются локально и синхронизируются автоматически.

QR-сканирование

Упростить инвентаризацию через QR-сканирование — быстрая идентификация запчасти без ручного ввода.

Аналитика для руководства

Дать руководству дашборд с графиками по остаткам и движениям, PDF/Excel отчёты для принятия решений.

Решение

Архитектура

Vanilla JS (ESM) + Vite Без фреймворков — минимальный размер бандла, быстрая загрузка даже на слабом соединении.

Firebase Firestore Реалтайм-синхронизация между всеми пользователями. Данные обновляются мгновенно.

Dexie (IndexedDB) Полный офлайн-режим: все операции записываются локально, автоматический sync при подключении.

Функционал

PWA Установка на домашний экран, service worker, cache-first стратегия.

QR-стикеры Генерация и сканирование для быстрой идентификации запчастей.

Chart.js + jsPDF + SheetJS Дашборд с графиками, экспорт отчётов в PDF и Excel.

Стек технологий

Frontend:
  Vanilla JS (ESM) + Vite
  Chart.js — дашборд с графиками
  jsPDF — экспорт в PDF
  SheetJS (XLSX) — экспорт в Excel
  QRcode — генерация и сканирование QR

Backend / Cloud:
  Firebase Firestore — реалтайм-база
  Firebase Storage — хранение фото
  Firebase Auth — аутентификация

Offline:
  Dexie (IndexedDB) — локальная база
  Service Worker — cache-first стратегия
  PWA — установка, офлайн-режим

UI:
  CSS (glassmorphism, адаптив)
  Ролевая модель: guest / engineer / admin
  PIN-авторизация
  Журнал действий (audit log)

Результаты и выводы

Ключевой вклад:

Проект демонстрирует способность создать полноценный цифровой продукт с нуля — от архитектуры до деплоя. Покрытие 3 складов и 44 залов, полный офлайн-режим, ролевая модель и аналитика для руководства.

  • 3 склада, 44 зала под управлением — единая система учёта для всего предприятия.
  • Офлайн-режим: работа без потери данных при отсутствии связи, автоматическая синхронизация.
  • Ролевая модель: guest / engineer / admin с PIN-авторизацией.
  • PDF/Excel отчёты: для руководства, дашборд с графиками.
  • Журнал действий: все операции привязаны к пользователю, полная прозрачность.
