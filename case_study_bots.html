Прототип Telegram-бота для управления задачами в Excel

Разработка и демонстрация (Proof-of-Concept) системы для двусторонней связи между Telegram и локальным Excel-файлом.

Telegram Bot API Python Excel (Локально) Автоматизация

Задача и Концепция

На производстве часто возникает потребность в быстрой постановке и отслеживании задач "в полях", без доступа к стационарному ПК. Этот проект был разработан как пилотный (Proof-of-Concept) для демонстрации возможности создания простой и эффективной системы таск-менеджмента.

Цель: Создать Telegram-бота, который может удаленно читать и изменять данные в локальном Excel-файле на компьютере, позволяя пользователям через телефон назначать задачи, менять их статусы и указывать ответственных.

Демонстрация работы

Прототип был успешно реализован и протестирован локально. Видео выше демонстрирует основной функционал:

Интерфейс Telegram-бота

  • Выбор специалиста (папки) из списка.
  • Просмотр существующих задач (протоколов).
  • Создание новой задачи по шаблону.
  • Редактирование задачи: изменение статуса ("В процессе", "Выполнено").
  • Редактирование задачи: назначение/смена ответственного.

Синхронизация с Excel

  • Локальный Excel-файл ("Автоматизация учета") выступает в роли базы данных.
  • Изменения, внесенные через бота (например, смена статуса на "Выполнено"), мгновенно отображаются в Excel-файле.
  • Изменения, внесенные вручную в Excel, также считываются ботом при следующем запросе.

Техническая реализация

Стек технологий

Для реализации прототипа использовались Python-скрипты, которые выполняли роль связующего звена (middleware) между облачным API Telegram и локальным файлом.

  • Telegram Bot API: Для приема команд от пользователя (кнопки, текст).
  • Python (aiogram/telebot): Библиотека для обработки логики бота.
  • Pandas/OpenPyXL: Библиотеки для чтения и записи данных в `.xlsx` файл.
  • Локальный сервер: Скрипт, запущенный на ПК, который "слушает" бота и имеет доступ к файловой системе.

Пример псевдокода (Логика)

Упрощенная логика обработчика изменения статуса задачи: 

# Псевдокод Python
@bot.callback_handler(func=lambda call: call.data.startswith('status_'))
def change_status(call):
    task_id = call.data.split('_')[1]
    new_status = call.data.split('_')[2]

    # 1. Найти Excel файл
    workbook = load_workbook('tasks.xlsx')
    sheet = workbook.active

    # 2. Найти нужную строку (задачу)
    row = find_task_by_id(sheet, task_id)

    # 3. Изменить статус в ячейке
    sheet[f'C{row}'] = new_status
    workbook.save('tasks.xlsx')

    # 4. Ответить пользователю
    bot.answer_callback_query(call.id, "Статус изменен!")

Результаты и выводы

Ключевой вклад:

Проект успешно доказал, что интеграция между облачными сервисами (Telegram) и локальными системами (Excel) возможна и эффективна. Это демонстрирует способность быстро создавать прототипы (R&D) и находить нестандартные решения для автоматизации рабочих процессов, используя современные инструменты (ИИ-ассистенты, API).

Этот прототип легко масштабируется путем замены локального Excel на облачные Google Таблицы (Google Sheets API), что позволило бы создать полноценную многопользовательскую систему таск-менеджмента без необходимости в локальном сервере.
